# 42_支出サマリー表示（A06_82）

## 機能概要

秘書×月のピボット形式で支出サマリーを表示する機能です。会計年度（4月開始）の12ヶ月分の支出額を一覧表示し、月別の列合計と行別の合計、総合計を確認できます。

## 実装状況

**実装済み**

## URL一覧

| HTTPメソッド | URL | 画面/処理名 | 説明 |
|---|---|---|---|
| GET | `/admin/summary/costs?fy={年度}` | 支出サマリー表示 | 秘書×月の支出をピボット表示 |

## 画面フロー

```
[管理者ホーム画面] または [ナビゲーションメニュー]
    ↓ 支出サマリーリンククリック
    ↓ GET /admin/summary/costs?fy={年度}
    ↓ クエリパラメータで制御：
    |   - fy: 会計年度（4月開始、省略時は今日から推定）
    ↓
[支出サマリー画面] (costs.jsp)
    - 会計年度表示（例: 2025年度 = 2025-04 ～ 2026-03）
    - ピボットテーブル
      - 行: 秘書名
      - 列: 12ヶ月（4月～翌3月）
      - セル: 支出金額（秘書への支払額）
      - 行合計: 各秘書の年間支出
      - 列合計: 各月の全秘書支出
      - 総合計: 年間総支出
    ↓
    └─ 年度変更: プルダウンで年度を選択して表示更新
```

## バックエンド処理詳細

### サービスクラス

- **クラス**: [SalesCostSummaryService.java](../../../src/main/java/service/SalesCostSummaryService.java)
- **メソッド**: `costSummary()` (136行目): 支出サマリー表示

### 処理フロー

#### 1. 支出サマリー表示（GET `/admin/summary/costs`）

1. **パラメータ取得・会計年度決定**
   - `fy`: 会計年度（4月開始）
   - 省略時または不正値の場合: 今日から推定
     - 4～12月: その年がFY
     - 1～3月: 前年がFY
   - `resolveFiscalYear()` ヘルパーメソッドで処理（178行目）

2. **12ヶ月のリスト生成**
   - `buildFiscalMonths(fy)` で対象FYの12ヶ月を生成（197行目）
   - 例: FY=2025 → ["2025-04", "2025-05", ..., "2026-03"]
   - yyyy-MM形式の文字列リスト

3. **期間範囲の確定**
   - `fromYm`: months[0]（最初の月、例: 2025-04）
   - `toYm`: months[11]（最後の月、例: 2026-03）

4. **支出データ取得**
   - `SecretaryMonthlySummaryDAO.selectCostsBySecretaryMonth(fromYm, toYm, months)` を呼び出し
   - `secretary_monthly_summaries` テーブルから集計
   - 秘書×月の支出をピボット形式で取得
   - 戻り値: `List<PivotRowDTO>`
     - 各行は1秘書を表す
     - `label`: 秘書名
     - `amountByYm`: Map<年月, 金額>（12ヶ月分）
     - `rowTotal`: 行合計

5. **列合計・総合計の計算**
   - `calcColTotals(months, rows)` で各月の列合計を計算（214行目）
   - `calcGrand(colTotals)` で総合計を計算（232行目）
   - Map<年月, 金額> と BigDecimal で管理

6. **画面用属性設定**
   - `fy`: 会計年度（int）
   - `months`: 12ヶ月のリスト（List<String>）
   - `rows`: ピボット行（List<PivotRowDTO>）
   - `colTotals`: 列合計（Map<String, BigDecimal>）
   - `grandTotal`: 総合計（BigDecimal）

7. **ビューへフォワード**
   - 支出サマリービュー（`summary/admin/costs`）へフォワード

### トランザクション管理

```java
public String costSummary() {
    try (TransactionManager tm = new TransactionManager()) {
        final int fy = resolveFiscalYear(req.getParameter(P_FY));
        final List<String> months = buildFiscalMonths(fy);

        final String fromYm = months.get(0);
        final String toYm   = months.get(months.size() - 1);

        // DAO 呼び出し：秘書×月のコストピボット
        SecretaryMonthlySummaryDAO dao = new SecretaryMonthlySummaryDAO(tm.getConnection());
        List<PivotRowDTO> rows = dao.selectCostsBySecretaryMonth(fromYm, toYm, months);

        Map<String, BigDecimal> colTotals = calcColTotals(months, rows);
        BigDecimal grand = calcGrand(colTotals);

        req.setAttribute(A_FY, fy);
        req.setAttribute(A_MONTHS, months);
        req.setAttribute(A_ROWS, rows);
        req.setAttribute(A_COLTOTAL, colTotals);
        req.setAttribute(A_GRAND, grand);

        return VIEW_COSTS;

    } catch (RuntimeException e) {
        validation.addErrorMsg("データベースに不正な操作が行われました");
        req.setAttribute("errorMsg", validation.getErrorMsg());
        return req.getContextPath() + req.getServletPath() + "/error";
    }
}
```

**特徴**:
- 読み取り専用処理のため、明示的なコミット不要
- try-with-resourcesで自動クローズ
- エラー時はエラーページへリダイレクト

### DAO呼び出し

- **SecretaryMonthlySummaryDAO**
  - `selectCostsBySecretaryMonth(String fromYm, String toYm, List<String> months)`: 支出ピボット取得（190行目）
    - SQL: `SQL_COSTS_BY_SECRETARY_MONTH`（53行目）
    - JOIN: `secretary_monthly_summaries → secretaries`
    - WHERE: `deleted_at IS NULL AND target_year_month BETWEEN ? AND ?`
    - GROUP BY: `secretary_id, name, target_year_month`
    - ORDER BY: `name, target_year_month`
    - Java側でピボット整形
      - 秘書IDごとに行を作成
      - 存在しない月は0で埋める
      - 行合計を計算

### セキュリティ

- **セッションチェック**: FrontControllerで管理者権限確認済み
- **論理削除フィルタ**: 削除済み秘書・サマリーデータは表示されない（`deleted_at IS NULL`）
- **会計年度の範囲制限**: 年度パラメータは2000～2100年の範囲で入力可能（JSP側で制限）

## 備考

### 会計年度の定義

- **4月開始**: 会計年度は4月1日～翌3月31日
- **年度の表記**: FY=2025 は 2025-04 ～ 2026-03 を意味する

### 支出の集計元

- **secretary_monthly_summaries テーブル**: 秘書月次サマリーテーブル
- 承認済みタスクを集計したデータ（他の機能で自動生成）
- 秘書への支払額（`total_secretary_amount`）を使用

### ピボット形式の利点

- 12ヶ月のトレンドを一目で確認
- 秘書別の年間支出を把握
- 月別の支出合計を比較

### 売上サマリーとの違い

| 項目 | 売上サマリー | 支出サマリー |
|-----|------------|------------|
| **軸** | 顧客 | 秘書 |
| **金額** | 顧客への請求額 | 秘書への支払額 |
| **元テーブル** | customer_monthly_invoices | secretary_monthly_summaries |
| **利益計算** | 売上 - 支出 = 利益（月別・年間） |

### 画面の特徴

- **横スクロール対応**: 12ヶ月+行合計の13列を表示
- **フォーマット**: 金額は3桁カンマ区切り（`#,##0`）
- **色分け**: Bootstrap の primary カラーでヘッダー・フッターを強調
- **売上サマリーとの並行表示**: 両画面を比較して利益を確認

### 対象データの範囲

- 論理削除されていない秘書・サマリーデータのみ表示
- 期間内にサマリーデータがない秘書は表示されない

### 使用場面

- 年間支出の確認
- 秘書別支出の比較
- 月別支出トレンドの確認
- 利益計算のための原価確認
- 秘書への支払予測

### 利益分析の方法

1. **売上サマリー**と**支出サマリー**を並べて表示
2. 各月の列合計を比較: 売上[月] - 支出[月] = 利益[月]
3. 総合計を比較: 売上合計 - 支出合計 = 年間利益
4. 秘書別・顧客別の収益性分析も可能
